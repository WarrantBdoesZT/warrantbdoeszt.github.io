📐 Foundational Principles

The DoD Zero Trust Reference Architecture defines five core tenets that govern every architectural decision.

1

Assume Hostile Environment

Malicious actors exist both inside and outside the network. All users, devices, and environments are treated as untrusted.

2

Presume Breach

Operate with the assumption an adversary already has presence. Enhanced scrutiny of all access and authorization decisions.

3

Never Trust, Always Verify

Deny by default. Every user, device, application/workload, and data flow is authenticated and explicitly authorized using least privilege.

4

Scrutinize Explicitly

All resources accessed securely using multiple dynamic and static attributes to derive confidence levels for contextual access.

5

Apply Unified Analytics

Unified analytics for DAAS (Data, Applications, Assets, Services) including behavioral analysis and logging of every transaction.

🗺️ DoD Zero Trust Capability Execution Roadmap

The DoD ZT Strategy mandates all components reach Target Level maturity by FY2027. This roadmap outlines the phased approach defined in the Capability Execution Roadmap (COA 1).

FY2022–FY2023 · Phase 0: Foundation
Establish Governance & Baseline
Stand up ZT program management, conduct capability gap assessments against the 7 pillars, and establish the enterprise identity foundation.
  • Appoint Zero Trust lead and governance body
  • Complete pillar-level gap assessment (152 activities)
  • Deploy enterprise MFA and ICAM foundation
  • Establish baseline network visibility and logging
FY2024–FY2025 · Phase 1: Initial ZT Capabilities
Deploy Core ZT Controls
Implement continuous authentication, device posture assessment, initial micro-segmentation, and automated policy enforcement.
  • Continuous multi-factor authentication (CMFA)
  • Device compliance and health attestation (Comply-to-Connect)
  • Macro-segmentation of network enclaves
  • Initial data tagging and classification automation
  • SIEM integration for cross-pillar visibility
FY2026–FY2027 · Phase 2: Target Level (Goal)
Achieve Target Level Maturity
All DoD components expected to reach Target Level ZT capability across all 7 pillars by end of FY2027.
  • Dynamic, risk-based access decisions in real time
  • Full micro-segmentation with software-defined perimeters
  • Automated SOAR response across the enterprise
  • Comprehensive data-at-rest, in-transit, and in-use protection
  • Behavioral analytics with AI/ML threat detection
FY2028+ · Phase 3: Advanced / Optimal
Advance to Optimal Maturity
Beyond FY2027 Target Level, components advance toward Optimal maturity with fully autonomous ZT operations and AI-driven policy.
  • Fully automated, self-healing ZT environment
  • AI/ML-driven dynamic policy generation
  • Full data lineage and automated rights management
  • Continuous compliance and automated remediation

🔧 Implementation Phases

A practitioner's phased approach to ZTA implementation, drawn from NIST SP 1800-35 and DoD ZT guidance.

Phase 1

Assess & Plan

Months 1–3
  • Inventory all assets (DAAS: Data, Apps, Assets, Services)
  • Map data flows and identify crown-jewel data
  • Conduct gap assessment against 7 pillars
  • Define protect surfaces (NIST SP 800-207)
  • Establish governance and stakeholder alignment
  • Develop ZT implementation roadmap with milestones
Phase 2

Foundation Building

Months 4–9
  • Deploy/upgrade IAM and ICAM foundation
  • Implement MFA across all user access paths
  • Deploy device compliance and posture assessment
  • Establish baseline network macro-segmentation
  • Deploy enterprise logging and SIEM visibility
  • Begin data classification and tagging
Phase 3

Advanced Capabilities

Months 10–18
  • Implement continuous multi-factor auth (CMFA)
  • Deploy micro-segmentation and SDP
  • Automate policy enforcement (SOAR)
  • Integrate behavioral analytics
  • Implement data-at-rest encryption and DRM
  • Establish transaction-level logging
Phase 4

Optimize & Sustain

Months 19+
  • AI/ML-driven dynamic policy decisions
  • Continuous compliance monitoring
  • Automated remediation workflows
  • Regular red-team validation of ZT controls
  • Advance pillar maturity toward Optimal
  • Expand ZT to all environments (cloud, tactical edge)

📐 ZTA Implementation Approaches

The NCCoE worked with 24 industry collaborators to build and document real ZTA implementations. These approaches represent validated deployment patterns DoD practitioners can emulate:

SDP / ZTNA
Software-Defined Perimeter and Zero Trust Network Access — identity-aware, per-application access broker
Enhanced Identity Governance
Identity-driven access model where identity is the primary policy enforcement point (EIG)
Microsegmentation
Granular network segmentation down to individual workloads with per-flow policy
Network Infrastructure
SDN-based overlays, inline CASB, and SWG integrated with identity-aware policy

🔗 Key Mappings: DoD ZT RA ↔ NIST SP 800-207 ↔ NIST SP 1800-35

DoD ZT RA Pillar NIST SP 800-207 Concept NIST SP 1800-35 Implementation
User Policy Engine (PE) identity inputs EIG, ICAM, MFA, conditional access policies
Device Device health posture feeds MDM, Comply-to-Connect, endpoint detection
Network/Environment Protect surface, SDP Microsegmentation, ZTNA, SDN overlays
Application & Workload Resource access via policy administrator Secure API gateways, CI/CD security, container security
Data Data-centric protection (SP 800-207 §3.2) Classification, encryption, DRM, DLP
Visibility & Analytics Trust algorithm inputs, continuous evaluation SIEM, UEBA, telemetry correlation
Automation & Orchestration Policy Engine automated decisions SOAR, automated response, policy-as-code

✅ ZT Readiness Checklist

Click items to track your progress. This checklist covers the foundational requirements across all 7 pillars for Target Level maturity.

👤 User Pillar

✓MFA deployed for all person and non-person entities
✓Continuous multi-factor authentication (CMFA) implemented
✓RBAC and ABAC policies defined and enforced
✓Federated identity established for partner orgs
✓Activity patterns continuously monitored

💻 Device Pillar

✓All devices inventoried and authenticated
✓Comply-to-Connect or MDM deployed
✓Real-time device posture assessment at access time
✓Automated remediation for non-compliant devices
✓Device health feeds into access policy decisions

🌐 Network/Environment Pillar

✓Macro-segmentation completed across enclaves
✓Micro-segmentation deployed for critical workloads
✓Software-defined perimeter (SDP) operational
✓Internal and external data flows mapped and controlled
✓Lateral movement paths identified and restricted

📦 Application & Workload Pillar

✓Application inventory and classification complete
✓Secure CI/CD pipeline with security scanning
✓Container and VM security controls deployed
✓Runtime workload protection active
✓API gateway with identity-aware policy enforcement

🔒 Data Pillar

✓Data classification and tagging scheme operational
✓Encryption at-rest for all sensitive data
✓Encryption in-transit (TLS) enforced enterprise-wide
✓Digital Rights Management (DRM) for controlled data
✓Data Loss Prevention (DLP) deployed

👁️ Visibility & Analytics Pillar

✓Enterprise SIEM with cross-pillar telemetry
✓UEBA (User & Entity Behavior Analytics) deployed
✓Transaction-level logging for all access events
✓Anomaly detection with alerting
✓Real-time dashboards for ZT posture monitoring

⚙️ Automation & Orchestration Pillar

✓SOAR platform deployed and integrated with SIEM
✓Automated incident response playbooks defined
✓Policy-as-code for consistent enforcement
✓Automated provisioning/deprovisioning of access
✓Continuous compliance monitoring with auto-remediation
